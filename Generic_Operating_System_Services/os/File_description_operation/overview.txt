                File Descriptor Operations  (Операции с файловым дескриптором)

Эти функции работают с потоками ввода-вывода, на которые ссылаются файловые дескрипторы.

Дескрипторы файлов — это маленькие целые числа, соответствующие файлу, открытому текущим
процессом. Например, стандартный ввод обычно имеет файловый дескриптор 0, стандартный
вывод — 1, а стандартная ошибка — 2. Последующим файлам, открытым процессом, будут
присвоены 3, 4, 5 и т. д. Название «файловый дескриптор» немного обманчиво; на платформах
Unix на сокеты и каналы также ссылаются файловые дескрипторы.

Метод fileno() можно использовать для получения дескриптора файла, связанного с файловым
объектом, когда это необходимо. Обратите внимание, что при непосредственном использовании
файлового дескриптора методы файлового объекта игнорируются, игнорируя такие аспекты,
как внутренняя буферизация данных.
